Getting Started
 
 
Steps

Login PERCEPTRON-XFMS as a guest (without user account) or register for PERCETRON-XFMS user account to login using an existing email account.

Step 1 – Input Parameters
save_alt
  • Go to the ‘XFMS Search Query’ in the menu on left and enter ‘XFMS Search Title’ to start data analysis.
  • Browse for the zipped folder containing following input data files:
      - Protein sequence file in standard .fasta* ** format.
      - Protein structure file in standard .pdb** format.
      - Mascot file in .xlsx format.
      - SASA** file in ‘.xlsx’ format.
      - DoseResposeInfo.txt files that contain the information regarding the X-ray doses used in experimental protocol.
Note: Users logging in via Guest Account can provide email address to receive results on Email ID upon search completion.
Step 2 – Upload Experimental Data
save_alt
  • For Replicate* 1: Browse for the folder containing Agilent LC-MS “.d” data files or .mzXML files . Add a zipped folder with multiple files (zipped .d or .mzXML files) of a single protein to search.
  • Repeat the same step for Replicate 2 and Replicate 3.
PERCEPTRON-XFMS requires three replicates for each sample data. Names of “*.d” files or “*.mzXML” files must strictly follow the same naming structure i.e., “Dose0.d”, “Dose10.d” or “Dose0.mzXML” etc.
Note: Please do not refresh the page or close the tab in middle of data uploading.
Step 3 – Protein-Water Contacts Analyses
save_alt
  • In order to analyze protein-water hydrogen bond graphs click the ‘Bridge Algorithm’ box.
  • Click on ‘Frustratometer Algorithm’ box for additional analysis of protein residue energetics.

Submit the Query Search by clicking on ‘Submit’ button.

Naming Convention
save_alt

User must follow the naming convention as provided into the sample data. This naming convention must also be followed in the DoseResposeInfo.txt files. Furthermore, for inputing the three replicates for each experimental data. Names of “.d” or “.mzXML” files must also follow the same naming structure i.e., “Dose0.d”, “Dose10.d” or “Dose0.mzXML”, “Dose10.mzXML” etc.
